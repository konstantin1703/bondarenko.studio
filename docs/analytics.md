# Analytics setup — Bondarenko.studio

На сайте подготовлена обвязка под Яндекс.Метрику и события конверсии.

## Что уже есть в коде

В `assets/js/main.js` добавлены:

- автоматическая загрузка Яндекс.Метрики, если на странице есть ID счётчика;
- функция `window.bndTrack(goalName, params)`;
- безопасная очередь `window.bndAnalyticsQueue`, если ID ещё не добавлен;
- отправка целей через `ym(counterId, 'reachGoal', goalName, params)`.

Если ID счётчика не указан, сайт работает как обычно, а события не отправляются в Яндекс.

## Как подключить ID счётчика

После создания счётчика в Яндекс.Метрике добавьте на страницы в `<head>`:

```html
<meta name="yandex-metrika-id" content="YOUR_COUNTER_ID">
```

Где `YOUR_COUNTER_ID` — числовой ID счётчика из Яндекс.Метрики.

Альтернативный способ — задать ID глобально до подключения `main.js`:

```html
<script>
  window.BND_METRIKA_ID = 12345678;
</script>
```

Рекомендуемый способ для этого статического сайта — meta-тег.

## Базовые события

| Событие | Когда отправляется | Зачем нужно |
|---|---|---|
| `open_modal` | Пользователь открыл модальное окно заявки | Измерить интерес к заявке |
| `submit_form_success` | Форма успешно отправлена | Главная конверсия сайта |
| `submit_form_error` | Ошибка валидации, сети или Worker | Найти проблемы формы |
| `click_telegram_modal` | Клик по Telegram внутри модалки | Понять, кто выбирает Telegram вместо формы |
| `click_footer_telegram` | Клик по Telegram в футере | Измерить прямой контакт из футера |
| `click_footer_email` | Клик по email в футере | Измерить email-контакт |
| `click_footer_phone` | Клик по телефону в футере | Измерить телефонный контакт |
| `click_mobile_menu` | Открытие burger-меню | Проверить полезность мобильной навигации |
| `click_mobile_cta` | Клик `Обсудить проект` в мобильном меню | Измерить мобильную конверсию |
| `theme_toggle` | Переключение темы | Вторичное UX-событие |

## Какие цели создать в Яндекс.Метрике

Минимальный набор целей:

1. `open_modal`
2. `submit_form_success`
3. `submit_form_error`
4. `click_telegram_modal`
5. `click_footer_telegram`
6. `click_mobile_cta`

Дополнительно можно создать:

- `click_footer_email`
- `click_footer_phone`
- `click_mobile_menu`
- `theme_toggle`

## Проверка

После добавления ID счётчика:

1. Откройте сайт.
2. Откройте DevTools → Console.
3. Выполните:

```js
window.bndTrack('test_goal')
```

4. В Яндекс.Метрике проверьте, что цель дошла.
5. Затем проверьте реальные события:
   - открыть модалку;
   - отправить тестовую заявку;
   - кликнуть Telegram в модалке;
   - открыть burger-меню на мобильной ширине.

## Важно

События не должны ломать сайт. Если Яндекс.Метрика не загрузилась или ID не указан, форма, модалка, меню и футер продолжают работать как обычно.
