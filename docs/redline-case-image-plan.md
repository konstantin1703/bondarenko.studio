# RedLine case — image selection plan

Branch: `sprint-2-content-polish`
Case page: `portfolio/case-4-redline-media.html`

## Goal

Use only the strongest screenshots. The case should prove that RedLine is not just a YouTube channel, but a working media infrastructure: YouTube + Telegram + bot + admin workflow + user feedback.

## Selected screenshots

### 1. YouTube channel overview

Purpose:

- show channel name;
- show avatar;
- show banner;
- show visual style;
- show that the project has a public media shell.

Editing:

- crop browser chrome if possible;
- keep banner, avatar, name, handle and short description;
- remove unnecessary empty lower area.

Suggested file name:

`assets/img/portfolio/redline/redline-youtube-channel.webp`

## Screenshots to avoid as standalone images

- duplicate Telegram bot screens that show the same step twice;
- command list only, unless needed as a small supporting image;
- screenshots with too much empty desktop area;
- screenshots that show the same proof as a stronger screenshot.

## Recommended placement in case page

1. After hero: YouTube channel + Telegram channel as public shell.
2. In bot section: bot start + submit flow + security check.
3. In admin workflow section: admin group + user status.

## Final selection count

Ideal: 6 screenshots.
Maximum: 8 screenshots.

The case should feel like a proof of system, not a screenshot dump.
