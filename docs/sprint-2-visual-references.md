# Sprint 2 — Visual references and borrowed mechanics

Branch: `sprint-2-content-polish`

## Principle

Do not copy other studios directly. Borrow mechanics, composition ideas and visual language, then adapt them to Bondarenko.studio.

The target is not a generic service landing page. The target is a compact AI-assisted digital studio with premium dark visuals, editorial typography and a stronger sense of craft.

## Features worth borrowing

### 1. Editorial hero

Use a stronger first screen:

- oversized serif headline;
- short and confident studio positioning;
- clear CTA;
- secondary CTA for portfolio;
- subtle abstract visual element on the right side;
- background glow / gradient layer / fine grid.

Goal: the first screen should immediately feel like a studio, not a price-list page.

### 2. Studio system panel

Add a visual panel that shows the studio workflow:

- Strategy;
- Copy;
- Design;
- Build;
- Launch.

This can look like a small operating dashboard or command panel, not a literal screenshot.

Goal: show that the studio has a method and process.

### 3. Case wall instead of simple cards

Upgrade portfolio presentation:

- one large featured case;
- two smaller supporting cases;
- tags like Strategy / Copy / Design / HTML;
- short case logic: task → solution → output;
- honest labels for internal/demo/published cases.

Goal: portfolio should feel like proof of skill, not placeholders.

### 4. Premium dark depth

Keep dark theme but make it richer:

- radial glow spots;
- soft gradient edges;
- thin borders;
- subtle noise/texture;
- better contrast hierarchy;
- hover glow on cards.

Goal: visual depth without heavy effects or slow loading.

### 5. Marquee / studio ticker

Add a restrained horizontal ticker with studio capabilities:

- Websites;
- Copywriting;
- Visual packaging;
- Landing pages;
- Brand messaging;
- AI-assisted production.

Goal: add movement and studio energy without turning the site into a circus.

### 6. Service packages instead of only price cards

Keep prices, but group them into studio packages:

- Launch package;
- Copy package;
- Visual package;
- Website package.

Goal: the site sells systems and outcomes, not only small tasks.

### 7. Honest blog/studio notes

If articles are not ready, do not show fake links. Use a "Studio notes" section:

- upcoming notes;
- ideas in progress;
- topics about websites, copy, packaging and AI workflow.

Goal: look honest and professional while content is still being built.

## Proposed implementation order

1. Upgrade `/portfolio/` first — it is currently the weakest public page.
2. Then upgrade homepage portfolio block into a case wall.
3. Then improve hero with studio visual panel.
4. Then clean blog section into honest Studio Notes.
5. Then decide whether to add marquee/ticker.

## Visual boundary

Allowed:

- premium dark style;
- editorial typography;
- gradients;
- glow;
- subtle motion;
- structured visual panels.

Avoid:

- random cyberpunk decoration;
- too much neon;
- heavy 3D unless there is a clear reason;
- animations that hide content;
- effects that make the site slow or unreadable.
